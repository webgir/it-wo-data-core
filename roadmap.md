# Roadmap — IT-WO Data Core (IWDC)

Этот документ фиксирует эволюцию IWDC по версиям.
Цель — прозрачное развитие системы и контроль сложностей.

## v0.1 — Skeleton (ТЕКУЩАЯ)

- Создана структура проекта.
- Добавлены базовые файлы документации в `docs/`.
- Определены JSON Schema для основных сущностей (series, model, length, seo, meta).
- Созданы заглушки скриптов в `scripts/`.
- Добавлены CLI-заглушки в `tools/cli/`.
- Подготовлен контекст для Cursor в `.cursor/`.

## v0.2 — XLS → CSV

- Реализовать `scripts/xls-to-csv.mjs` с конфигом маппинга листов.
- Описать формат входного XLS в `docs/xls-mapping.md`.
- Определить целевые CSV (`models.csv`, `lengths.csv`, `seo.csv`, `texts.csv`).
- Добавить тестовый пример в `examples/tiny/`.

## v0.3 — CSV → JSON

- Реализовать `scripts/csv-to-json.mjs`.
- Описать маппинг CSV → JSON в `docs/csv-json.md`.
- Сгенерировать первый набор `data/json` для тестового примера.
- Обновить `docs/data-structures.md` примерными объектами.

## v0.4 — Базовая валидация

- Реализовать `scripts/validate.mjs` на базе JSON Schema.
- Описать доменные правила в `docs/validation.md`.
- Добавить базовые проверки связности (model_slug, длины, диапазоны).

## v0.5 — Diff + Changelog

- Реализовать `scripts/diff.mjs` (старый/новый JSON слой).
- Реализовать `scripts/changelog.mjs` (человекочитаемый отчёт).
- Описать формат отчётов в `docs/diff-changelog.md`.

## v0.6 — CI/CD и Quality Gates

- Добавить конфигурации для GitHub Actions (или другой CI-системы).
- Определить Quality Gates (валидация обязательна перед merge/deploy).
- Задокументировать в `docs/pipeline.md`.

## v0.7 — CLI и дев-удобство

- Дописать CLI-команды в `tools/cli/*`.
- Обновить `.cursor/commands.md` для работы в паре с Cursor.
- Описать стандартные сценарии запуска (dev, full, dry-run).

## v0.8 — Multilanguage

- Расширить схемы (локализации, поля `locale` и блоки для разных языков).
- Описать паттерн multilanguage в `docs/data-structures.md`.
- Добавить пример multi-lang данных в `examples/tiny/`.

## v0.9 — API-контракты

- Описать контракты API в `docs/api-contracts.md`.
- При необходимости — подготовить референс-реализацию API (отдельный проект).

## v1.0 — Стабильное ядро

- Все основные компоненты реализованы и покрыты тестами.
- Документация актуальна и непротиворечива.
- Проект готов к использованию как ядро B2B-экосистемы и к патентованию.

---

## Правила обновления Roadmap

- Любое изменение архитектуры — фиксация в `architecture.md` и `roadmap.md`.
- Любое добавление/изменение этапа — новая запись с датой и кратким комментарием.
