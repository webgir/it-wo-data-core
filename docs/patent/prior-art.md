Анализ уровня техники и известных решений (Prior Art)



Изобретение относится к методам обработки, контроля целостности и синхронизации структурированных данных. На дату подачи заявки в области управления данными известен широкий спектр решений, включая:



1. Системы контроля версий (Git и аналоги)



Системы контроля версий позволяют сохранять изменения в файловой структуре, выполнять сравнение версий и работать с ветками. Однако данные системы:



— не выполняют многоуровневую валидацию структурированных данных;

— не обеспечивают предиктивный анализ целостности;

— не формируют структурированные diff для объектов данных;

— не поддерживают механизмы восстановления, ориентированные на структурированные объекты;

— не обеспечивают встроенной распределённой синхронизации состояния данных.



2. ETL-платформы (Extract–Transform–Load)



Современные ETL-системы выполняют преобразование данных между источниками и хранилищами, но:



— не обеспечивают строгой детерминированности состояния данных;

— не предоставляют встроенной системы snapshot/diff/changelog;

— не гарантируют сохранение структурированных семантических связей между объектами;

— не обладают встроенным механизмом восстановления состояния.



3. Базы данных с триггерами и журналированием



СУБД (PostgreSQL, MySQL, Oracle и др.) предоставляют средства журналирования транзакций, триггеров, ограничений и проверок. Однако:



— база данных не представляет собой структурированную файловую систему, на которую ориентирован данный способ;

— отсутствует машинно-читаемый diff между версиями набора данных;

— отсутствует концепция snapshot на уровне файловых структур;

— не предусмотрен предиктивный анализ целостности.



4. Системы документооборота и CMS



Системы управления контентом обеспечивают хранение документов, но:



— не выполняют глубокую валидацию структурных объектов;

— не предоставляют формальный diff между версиями;

— не поддерживают восстановление на уровне структурированных моделей данных;

— не реализуют распределённое согласование версий.



5. Системы DevOps/CI-CD



Инструменты автоматизации поставки (GitLab CI, GitHub Actions, Jenkins):



— не обеспечивают обработки структурированных данных;

— не выполняют предиктивные проверки данных;

— не включают механизмы восстановления структурированных наборов данных;

— не являются системами синхронизации состояния объектов.



Вывод о новизне



На момент подачи заявки не обнаружено решений, которые объединяли бы в рамках одного способа следующие элементы:



многоуровневую структурную, логическую, контекстную и обратную валидацию;



предиктивный анализ целостности данных до формирования версии;



формирование детерминированного snapshot структурированных объектов;



вычисление формализованного diff структурированных данных;



построение машинного и человекочитаемого changelog;



интегрированный механизм восстановления состояния;



распределённую синхронизацию состояния с выбором ведущего узла и предотвращением split-brain.



Такое сочетание признаков в известных решениях отсутствует, а потому предлагаемый способ обладает новизной, изобретательским уровнем и промышленной применимостью.
