Формула изобретения



1. Способ управления структурированной системой данных, включающий:



приём входных данных из одного или нескольких внешних источников;



преобразование входных данных в структурированный набор объектов в соответствии с предопределёнными контрактами данных;



выполнение многоуровневой валидации данных, включающей:

— структурную проверку соответствия объектов предопределённым схемам;

— проверку типов данных;

— проверку связей между объектами;

— логическую валидацию значений;

— проверку обратной совместимости с предыдущими версиями данных;



выполнение предиктивного анализа целостности данных перед формированием версии, включающего обнаружение потенциальных аномалий, несоответствий, нарушений связей и других признаков предстоящих ошибок;



формирование снимка состояния данных (snapshot), включающего сохранение полного набора объектов, метаданных версии, контрольных сумм и ссылки на предыдущую версию;



вычисление отличий (diff) между текущим снимком состояния данных и предыдущей версией, включающих добавления, удаления, изменения и нарушения совместимости;



автоматическое формирование журнала изменений (changelog) на основе вычисленных отличий;



выполнение восстановительных операций, включающих откат состояния данных к предыдущему снимку версии и восстановление корректности структуры данных;



ведение распределённого согласованного состояния системы данных, включающего выбор ведущего узла, обмен heartbeat-сообщениями, блокировки ресурсов, журналирование межузловых операций, обнаружение и разрешение конфликтов, а также синхронизацию версий данных;



предоставление актуального набора данных во внешние системы посредством программного интерфейса (API) в соответствии с контрактами данных.



Зависимые пункты



2. Способ по п. 1, отличающийся тем, что многоуровневая валидация включает автоматическую проверку уникальности идентификаторов объектов и непрерывности цепочек ссылок.



3. Способ по п. 1, отличающийся тем, что предиктивный анализ целостности включает анализ статистических и структурных аномалий данных.



4. Способ по п. 1, отличающийся тем, что формирование снимка состояния данных выполняется с использованием детерминированного алгоритма, обеспечивающего воспроизводимость результата при повторных выполненияx способа.



5. Способ по п. 1, отличающийся тем, что вычисление отличий между версиями данных включает анализ нарушений обратной совместимости.



6. Способ по п. 1, отличающийся тем, что восстановительные операции выполняются автоматически при обнаружении нарушений целостности данных.



7. Способ по п. 1, отличающийся тем, что распределённая синхронизация обеспечивает предотвращение состояния split-brain за счёт выбора единственного ведущего узла.



8. Способ по п. 1, отличающийся тем, что предоставление данных во внешние системы выполняется в виде детерминированного API, генерируемого на основе последнего успешного снимка данных.
