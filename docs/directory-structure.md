# IWDC — Directory Structure

Архитектура IWDC v0.1 основана на принципе изолированных слоёв: источники данных, промежуточные форматы, итоговые JSON и служебные инструменты разделены по своим уровням ответственности. Цель структуры — обеспечить предсказуемость, стандартизацию и автоматизируемость каталогов.

## 1. Корневая структура

it-wo-data-core/
  README.md
  roadmap.md
  docs/
  schemas/
  scripts/
  sources/
  intermediate/
  data/
  tools/
  examples/
  .cursor/

Каждая директория имеет строгое назначение и не должна содержать ручные файлы, если это не предусмотрено архитектурой.

## 2. docs/

Документационная зона. Здесь хранится полная архитектура IWDC, правила, пайплайн, соглашения, спецификации и глоссарий. Эти файлы служат базой знаний для разработчиков, Cursor и автоматизации.

## 3. schemas/

JSON Schema для всех сущностей IWDC: серии, модели, длины, мета-поля, SEO. Используются для строгой валидации данных и автоматического анализа контента.

## 4. scripts/

Скрипты-конвейеры, выполняющие:
– преобразование XLS → CSV  
– конвертацию CSV → JSON  
– валидацию  
– сравнение данных  
– генерацию changelog  

Все скрипты написаны как отдельные модули и не зависят от фронтенд-части.

## 5. sources/

Зона внешних исходников.  
sources/xls/ — XLS-файлы, полученные извне.  
Файлы в эту папку добавляет только пользователь или интеграция.

## 6. intermediate/

Промежуточное хранилище.
intermediate/csv/ — CSV-файлы, полученные после обработки XLS.  
Ничего ручного здесь быть не должно.

## 7. data/

Это итоговые данные IWDC, которые будут использованы в любых системах:

data/json/series/  
data/json/models/  
data/json/lengths/

По умолчанию директории содержат только .gitkeep. Все файлы JSON генерируются пайплайном и не создаются вручную.

## 8. tools/

CLI-инструменты IWDC.  
tools/cli/ включает команды:

iwdc-build.mjs  
iwdc-validate.mjs  
iwdc-diff.mjs  

Эти файлы вызывают соответствующие скрипты из /scripts.

## 9. examples/

Готовые минимальные рабочие примеры для обучения, демонстрации и тестирования конвейера.

Стандартный tiny-набор:
examples/tiny/tiny.xlsx  
examples/tiny/tiny-expected.json

## 10. .cursor/

Файлы, обеспечивающие корректную работу Cursor в рамках IWDC:

project-context.md  
rules.md  
commands.md

Они дают контекст, ограничения и инструкции для автоматизации работы с репозиторием.
