# IWDC — Glossary

Глоссарий содержит ключевые термины, используемые в архитектуре IWDC v0.1. Он определяет единый язык системы и обеспечивает согласованность между документацией, скриптами, CLI и схемами.

---

## 1. Series (серия)

Группа моделей, объединённых общей конструктивной или эксплуатационной логикой.  
Пример: "VK", "VKV24", "VKN".

Series — верхний уровень иерархии IWDC.

---

## 2. Model (модель)

Конкретная конфигурация оборудования, задаваемая артикулом.  
Пример: "VK.65.160.2TG".

Модель связывает серию и набор длиночных характеристик.

---

## 3. Length (длина)

Единица измерения мощности и стоимости оборудования.  
Каждая длина имеет:
– длину (мм)  
– теплоотдачу (Вт)  
– цену (₽)

В IWDC длина — базовый объект расчётов.

---

## 4. CSV

Промежуточный текстовый формат.  
Создаётся из XLS и содержит нормализованную структуру:
series, model, length, watts, price.

Это единственный допустимый формат для входа в JSON-пайплайн.

---

## 5. JSON Output

Итоговое представление модели IWDC:
{
  "series": "...",
  "model": "...",
  "lengths": [...]
}

JSON — конечный артефакт, используемый во внешних системах.

---

## 6. Pipeline

Конвейер обработки данных IWDC.  
Этапы: XLS → CSV → JSON → Validate → Diff → Changelog.  
Полностью автоматизирован, детерминирован и строг.

---

## 7. Validate

Система проверки корректности данных.  
Проверяет JSON Schema, структуру директорий, значения и согласованность.

Валидация — обязательный этап сборки.

---

## 8. JSON Schema

Формальное описание структуры данных (серий, моделей, длин).  
Используется для строгой проверки входных данных.

Схемы размещены в каталоге `/schemas`.

---

## 9. Diff

Система сравнения текущих и предыдущих данных.  
Используется для наблюдения изменений при обновлении XLS.

Генерирует структурированный отчёт.

---

## 10. Changelog

Документ, фиксирующий изменения в данных.  
Содержит:
– что изменилось  
– что было удалено  
– что добавилось

Используется при публикации.

---

## 11. Sources

Директория с исходными XLS-файлами.  
Определяет верхний слой ввода данных.

---

## 12. Intermediate

Промежуточный слой, содержащий CSV-файлы.  
Ничего кроме CSV здесь быть не должно.

---

## 13. Data

Итоговые JSON-файлы, используемые каталогами, API и внешними системами.  
Формируются автоматически.

---

## 14. CLI

Инструменты управления IWDC:
– iwdc-build  
– iwdc-validate  
– iwdc-diff

CLI — единая точка входа для запусков пайплайна.

---

## 15. Cursor Rules

Файлы внутри `.cursor/`, определяющие:
– контекст  
– правила  
– команды

Cursor использует их для корректной автоматизации IWDC.
