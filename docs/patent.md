## 2. Объект защиты

Система IWDC (Integrated Wilma Data Core) представляет собой модульную архитектуру обработки каталогов, включающую:

1. стандартизированные XLS → CSV → JSON преобразования;  
2. многоуровневую систему валидации;  
3. строгую файловую структуру;  
4. унифицированные JSON Schema;  
5. согласованную модель данных “series → model → length”;  
6. инфраструктуру CLI для автоматизации;  
7. алгоритмы дифференциального анализа данных (DIFF);  
8. генерацию структурированных changelog;  
9. tiny-пример как эталон минимальной модели;  
10. изолированный архитектурный слой для Cursor.

В совокупности компоненты образуют единую информационную систему, обладающую оригинальной архитектурой.

---

## 3. Архитектурная модель

IWDC использует многоуровневую модель данных:

– **Sources Layer** (XLS)  
– **Intermediate Layer** (CSV)  
– **Data Layer** (JSON)  
– **Validation Layer** (Schema-based)  
– **Automation Layer** (CLI)  
– **Documentation Layer** (docs/*)  

Каждый слой изолирован, что обеспечивает модульность, тестируемость и детерминированность результата.

---

## 4. Уникальные особенности системы

1. **Единый детерминированный pipeline**, гарантирующий идентичность результата.  
2. **Строгая привязка к JSON Schema**, исключающая любые отклонения в данных.  
3. **Автоматический DIFF**, позволяющий фиксировать изменения во внешнем XLS-файле.  
4. **Стандартизированное размещение файлов JSON**, что снижает вероятность ошибок.  
5. **Минимальный tiny-эталон**, позволяющий тестировать pipeline в изолированных условиях.  
6. **Слой интеграции для Cursor**, обеспечивающий предсказуемую автоматизацию.
